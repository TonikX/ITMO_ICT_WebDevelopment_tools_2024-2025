    Сравнение способов параллельного вычисления суммы чисел

    Описание программ

    threading_sum.py

    Программа вычисляет сумму чисел от 1 до limit с использованием нескольких потоков (модуля threading).
    Каждый поток обрабатывает свою часть диапазона чисел и сохраняет результат в общий список.
    В конце основной поток суммирует все промежуточные результаты.

    Особенности метода: - Использует потоки (threading.Thread); - Подходит для операций, не требующих высокой загрузки CPU; - Из-за GIL (Global Interpreter Lock) в Python потоки не дают значительного выигрыша в вычислениях, связанных с процессором.

    multiprocessing_sum.py

    Программа вычисляет сумму чисел от 1 до limit с использованием нескольких процессов (модуля multiprocessing).
    Каждый процесс независимо обрабатывает часть диапазона и записывает свой результат в общую Manager().dict().

    Особенности метода:

    • Использует процессы (multiprocessing.Process);
    • Каждый процесс работает в своей копии интерпретатора Python, минуя GIL;
    • Лучше подходит для задач, требующих высокой загрузки CPU.

    async_sum.py

    Программа вычисляет сумму чисел от 1 до limit с использованием асинхронного программирования (asyncio). В вычислениях используются несколько асинхронных задач, которые обрабатывают отдельные диапазоны чисел.

    Особенности метода: - Использует асинхронные задачи (asyncio) для параллельной обработки без использования потоков; - Включает паузу на await asyncio.sleep(0), чтобы предотвратить блокировку остальных задач во время долгих вычислений; - Подходит для операций с асинхронным выполнением, но может не продемонстрировать значительного ускорения для тяжёлых вычислений из-за GIL (Global Interpreter Lock); - Позволяет легко масштабировать количество задач, изменяя параметр num_tasks.

    Таблица сравнения времени выполнения

    Метод Время выполнения (секунд)
    Потоки (threading) 42.846
    Процессы (multiprocessing) 12.129
    Асинхронность (asyncio + потоки) 68.347

    Выводы по использованным методам

    • multiprocessing показал наилучший результат (12.129 сек). Такой результат возможен из-за того, что процессы работают независимо друг от друга, эффективно используя все ядра процессора. Это лучший подход для задач, сильно нагружающих CPU.
    • threading и asyncio оказались гораздо медленнее (42-68 сек), так как из-за GIL в Python потоки работают не параллельно на уровне процессора, а скорее переключаются между собой. Для CPU-вычислений они малоэффективны.
    • Асинхронность (asyncio) отлично подходит для задач с большим количеством операций ввода-вывода (например, сетевые запросы), но для математических вычислений она не даёт прироста производительности.

    Краткий итог:

    • Для тяжёлых вычислений лучше использовать multiprocessing;
    • К сожалению, потоки и асинхронность не ускоряют чистые математические расчёты в Python.
