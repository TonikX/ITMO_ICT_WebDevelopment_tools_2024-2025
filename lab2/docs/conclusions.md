## Ключевые выводы

### 1. Multiprocessing 🏆
**Лучший выбор для:**
- CPU-интенсивных задач (математические вычисления)
- Задач, требующих истинного параллелизма
- Случаев, когда нужно обойти ограничения GIL

**Показал лучшие результаты в обеих задачах**

### 2. Threading ⚠️
**Подходит для:**
- I/O операций (файлы, сеть, база данных)
- Задач с большим временем ожидания
- Простых случаев параллелизма

**Ограничения:** GIL препятствует истинному параллелизму для CPU-задач

### 3. Async/Await ⚡
**Идеален для:**
- Большого количества одновременных I/O операций
- Веб-серверов и API
- Задач с множественными ожиданиями

**Особенности:** Эффективен при правильном применении для I/O

## Практические рекомендации

### Выбор подхода по типу задачи:

1. **Математические вычисления, обработка данных** → **Multiprocessing**
2. **Веб-скрапинг, API-запросы** → **Async** или **Multiprocessing**
3. **Файловые операции** → **Threading** или **Async**
4. **Простые параллельные задачи** → **Threading**

### Факторы для учета:

- **Количество CPU ядер** - влияет на эффективность Multiprocessing
- **Тип операций** - CPU vs I/O интенсивные
- **Размер данных** - большие объемы лучше обрабатывать процессами
- **Сложность синхронизации** - Threading проще для разделяемых данных

## Заключение

Лабораторная работа наглядно продемонстрировала различия между подходами к параллелизму в Python. **Multiprocessing** показал стабильно высокие результаты для обеих задач, что подтверждает его универсальность для ресурсоемких операций. **Threading** и **Async** имеют свои ниши применения, особенно для I/O операций, но требуют более тщательного выбора в зависимости от специфики задачи.
