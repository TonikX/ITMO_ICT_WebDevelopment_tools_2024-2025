# Сравнение результатов - Задание 1

## Результаты тестирования

| Подход | Время выполнения | Количество workers | Ускорение |
|--------|-----------------|-------------------|-----------|
| Threading | 28.71 сек | 8 потоков | ~1x |
| Multiprocessing | 5.72 сек | 8 процессов | ~5x |
| Async | 28.22 сек | 8 задач | ~1x |

## Анализ результатов

### 🏆 Multiprocessing - Победитель (5.72 сек)
- **Преимущества:**
  - Обходит ограничения GIL
  - Использует все ядра CPU эффективно
  - Истинный параллелизм для CPU-задач
- **Ускорение:** В ~5 раз быстрее остальных подходов

### Threading (28.71 сек)
- **Ограничения:**
  - GIL не позволяет истинный параллелизм для CPU-задач
  - Потоки выполняются поочередно при математических вычислениях
- **Применение:** Лучше подходит для I/O операций

### Async (28.22 сек)
- **Ограничения:**
  - Работает в одном потоке
  - Кооперативная многозадачность не дает преимуществ для вычислений
- **Применение:** Эффективен для I/O операций с ожиданием

## Выводы

1. **Для CPU-интенсивных задач** (математические вычисления) - используйте **Multiprocessing**
2. **Threading и Async** не подходят для чистых вычислений из-за ограничений Python
3. **GIL** является основным ограничивающим фактором для Threading
4. **Multiprocessing** показал ожидаемые результаты - почти линейное ускорение по количеству ядер CPU
