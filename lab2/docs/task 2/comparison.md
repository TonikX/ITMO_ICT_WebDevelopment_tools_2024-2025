# Сравнение результатов - Задание 2

## Результаты тестирования

| Подход | Время выполнения | Количество workers | Ускорение |
|--------|-----------------|-------------------|-----------|
| Multiprocessing | 2.10 сек | 8 процессов | ~1.9x |
| Async | 3.23 сек | 8 задач | ~1.2x |
| Threading | 3.93 сек | 8 потоков | ~1x |

## Анализ результатов

### 🏆 Multiprocessing - Неожиданный победитель (2.10 сек)
- **Преимущества:**
  - Полный параллелизм HTTP-запросов
  - Каждый процесс независимо обрабатывает запросы
  - Нет конкуренции за ресурсы между процессами
- **Неожиданность:** Обычно для I/O задач async показывает лучшие результаты

### Async (3.23 сек)
- **Характеристики:**
  - Эффективное использование одного потока
  - Неблокирующие HTTP-запросы через aiohttp
  - Кооперативная многозадачность
- **Результат:** Средняя производительность

### Threading (3.93 сек)
- **Характеристики:**
  - GIL менее критичен для I/O операций
  - Потоки блокируются на сетевых запросах
  - Переключение контекста между потоками
- **Результат:** Самый медленный из трех

## Обработанные данные
- **Всего URL:** 8 страниц
- **Успешно обработано:** 8 страниц
- **Страниц с заголовками:** 1 (JSONPlaceholder)
- **Страниц без заголовков:** 7 (API endpoints httpbin.org)

## Выводы

1. **Multiprocessing** показал лучшие результаты даже для I/O задач в данном тесте
2. **Async** занял среднее место, что необычно для сетевых операций
3. **Threading** оказался самым медленным из-за overhead переключения потоков
4. **Результаты могут варьироваться** в зависимости от сетевых условий и характера запросов
5. **Для небольшого количества URL** разница в подходах менее критична
