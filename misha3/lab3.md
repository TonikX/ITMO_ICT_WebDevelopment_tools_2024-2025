# Архитектура системы обмена книгами с Celery и FastAPI

Проект представляет собой систему обмена книгами между пользователями, состоящую из нескольких сервисов, управляемых через Docker Compose:

## Сервисы

- **db** – PostgreSQL база данных для хранения пользователей, книг, предложений обмена.
- **redis** – брокер сообщений для Celery и кэширование.
- **books** – основной FastAPI-приложение для управления книгами и пользователями.
- **parser** – сервис для парсинга HTML страниц с сайта books.toscrape.com.
- **celery_worker** – воркер Celery для асинхронной обработки задач парсинга.

---

## Модели данных

### Основные сущности:
- **User** – пользователи системы с аутентификацией
- **Book** – каталог книг (название, автор, жанр, год издания)
- **UserBook** – связь пользователя с книгой (статус: доступна/заказана/обменена)
- **Offer** – предложения обмена между пользователями
- **Exchange** – завершённые обмены книгами

---

## Celery задачи

### 1. `parse_url_tasks`

- Асинхронно парсит одну страницу с сайта books.toscrape.com.
- Использует `httpx.AsyncClient` для загрузки HTML.
- Отправляет HTML на сервис `parser` для извлечения данных о книгах.

### 2. `parse_all_urls`

- Асинхронно обрабатывает список URL из `urls`.
- Вызывает `_parse_url` для каждого URL и собирает результаты.

## FastAPI роуты

### Управление пользователями (`/users`)
- `POST /users/register` – регистрация нового пользователя
- `POST /users/login` – аутентификация пользователя
- `GET /users/me` – получение текущего пользователя
- `GET /users/` – список всех пользователей

### Управление книгами (`/books`)
- `POST /books/` – добавление новой книги
- `GET /books/` – список всех книг
- `GET /books/{book_id}` – получение книги по ID
- `PUT /books/{book_id}` – обновление книги
- `DELETE /books/{book_id}` – удаление книги

### Управление предложениями обмена (`/offers`)
- `POST /offers/` – создание предложения обмена
- `GET /offers/` – список предложений
- `GET /offers/{offer_id}` – получение предложения по ID
- `PUT /offers/{offer_id}` – обновление предложения
- `DELETE /offers/{offer_id}` – удаление предложения

### Парсинг книг (`/parse-*`)
- `POST /parse-url` – парсинг одной страницы (параметр: `url`)
- `POST /parse-all` – парсинг всех URL из списка
- `GET /task-status/{task_id}` – проверка статуса задачи парсинга

### Управление обменами (`/exchanges`)
- Роуты для управления завершёнными обменами

## Особенности

- **Асинхронность**: Использование `asyncio` и `httpx` для неблокирующих операций
- **Celery**: Redis как брокер и backend для фоновых задач
- **Микросервисная архитектура**: Парсер работает в отдельном сервисе
- **Аутентификация**: JWT токены для защиты API
- **База данных**: PostgreSQL с миграциями через Alembic
- **ORM**: SQLModel для работы с базой данных
