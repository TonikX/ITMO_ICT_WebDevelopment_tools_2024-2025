# Документация Time Manager API

## Обзор

Time Manager API - это приложение на базе FastAPI для управления задачами, учета времени и назначений пользователей. Система обеспечивает функциональность аутентификации пользователей, управления задачами, отслеживания времени и назначения задач на основе ролей.

## Основные возможности

- Аутентификация пользователей с помощью JWT
- Управление задачами
- Учет времени
- Распределение задач на основе ролей
- Управление пользователями

## Технический стек

- **Фреймворк**: FastAPI
- **База данных**: PostgreSQL
- **ORM**: SQLModel
- **Миграции**: Alembic

## Модели данных

### Пользователь (User)

- `id`: Первичный ключ
- `username`: Уникальное имя пользователя
- `email`: Уникальный email
- `hashed_password`: Зашифрованный пароль
- `bio`: Опциональная биография пользователя

### Задача (Task)

- `id`: Первичный ключ
- `title`: Название задачи (индексировано)
- `description`: Опциональное описание задачи
- `deadline`: Опциональный срок выполнения
- `priority`: Уровень приоритета (1-высокий, 2-средний, 3-низкий)
- `status`: Статус задачи (Ожидает, В процессе, Завершена)
- `creator_id`: Ссылка на создателя задачи

### Учет времени (Time Entry)

- `id`: Первичный ключ
- `task_id`: Ссылка на задачу
- `user_id`: Ссылка на пользователя
- `duration`: Продолжительность в минутах
- `entry_date`: Дата создания записи
- `comment`: Опциональный комментарий

### Назначение задачи (Task Assignment)

- `id`: Первичный ключ
- `task_id`: Ссылка на задачу
- `user_id`: Ссылка на пользователя
- `role`: Роль пользователя в задаче

## API Endpoints

### Управление пользователями

```http
POST /users/register - Регистрация нового пользователя
POST /users/login - Аутентификация пользователя
GET /users - Список всех пользователей
GET /users/{user_id} - Получение данных пользователя
PUT /users/{user_id}/password - Обновление пароля пользователя
```

### Управление задачами

```http
POST /tasks - Создание новой задачи
GET /tasks - Список всех задач
GET /tasks/{task_id} - Получение данных задачи
PUT /tasks/{task_id} - Обновление задачи
DELETE /tasks/{task_id} - Удаление задачи
```

### Учет времени

```http
POST /time-entries - Создание записи времени
GET /time-entries - Список записей времени пользователя
PUT /time-entries/{entry_id} - Обновление записи времени
DELETE /time-entries/{entry_id} - Удаление записи времени
```

### Назначения задач

```http
POST /assignments - Назначение пользователя на задачу
GET /assignments - Список всех назначений
```

## Аутентификация

API использует JWT (JSON Web Token) для аутентификации. Для доступа к защищенным endpoint'ам:

1. Войдите через `/users/login` для получения токена доступа
2. Включите токен в заголовок запроса:

```http
Authorization: Bearer <ваш_jwt_токен>
```

## Конфигурация базы данных

Конфигурация базы данных управляется через переменные окружения:

- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `POSTGRES_DB`
- `POSTGRES_SERVER`
- `POSTGRES_PORT`

## Функции безопасности

- Хеширование паролей с использованием SHA256
- Аутентификация через JWT токены
- Контроль доступа на основе ролей
- Срок действия токенов
- Защищенные endpoint'ы
