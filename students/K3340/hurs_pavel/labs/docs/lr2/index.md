# Задача 1: Параллельное вычисление суммы чисел

## Описание задачи

Реализация вычисления суммы чисел от 0 до N с использованием различных подходов параллельного программирования.

## Реализованные подходы

### 1. Threading Implementation

- Использует потоки для параллельных вычислений
- Реализовано в классе [`ThreadSummation`](summation/implementations/threads.py)
- Использует блокировку (`threading.Lock`) для безопасного обновления общей суммы
- Каждый поток обрабатывает свой диапазон чисел

### 2. Multiprocessing Implementation

- Использует процессы для параллельных вычислений
- Реализовано в классе [`ProcessSummation`](summation/implementations/processes.py)
- Использует `multiprocessing.Pipe` для передачи результатов между процессами
- Каждый процесс вычисляет сумму своего диапазона независимо

### 3. Asyncio Implementation

- Использует асинхронное программирование
- Реализовано в классе [`AsyncSummation`](summation/implementations/async_impl.py)
- Создает асинхронные задачи для каждого диапазона
- Использует `asyncio.gather` для сбора результатов

## Результаты тестирования

- Все три подхода корректно вычисляют общую сумму
- График производительности доступен в [summation.png](summation.png)
- Сравнительный анализ времени выполнения для разных размеров входных данных

## Структура проекта

```
summation/
├── __init__.py
├── abstractions.py    # Абстрактный базовый класс
├── benchmark.py       # Код для тестирования производительности
├── main.py           # Основной файл запуска
├── utils.py          # Вспомогательные функции
└── implementations/  # Реализации различных подходов
```

# Задача 2: Параллельный парсинг веб-страниц

## Описание задачи

Реализация параллельного парсинга заголовков веб-страниц с сохранением результатов в SQLite базу данных.

## Реализованные подходы

### 1. Threading Implementation

- Использует потоки для параллельного парсинга
- Каждый поток обрабатывает отдельный URL
- Общая база данных с синхронизацией доступа

### 2. Multiprocessing Implementation

- Использует процессы для параллельного парсинга
- Каждый процесс работает с отдельным URL
- Реализована передача данных между процессами

### 3. Asyncio Implementation

- Асинхронный парсинг веб-страниц
- Использует aiohttp для HTTP-запросов
- Эффективная обработка I/O операций

## База данных

- Используется SQLite с WAL режимом
- Схема таблицы:
  ```sql
  CREATE TABLE pages (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      url TEXT UNIQUE,
      title TEXT
  )
  ```
- Реализована защита от конкурентного доступа

## Структура проекта

```
webpages/
├── __init__.py
├── abstractions.py    # Абстрактный базовый класс
├── benchmark.py       # Тестирование производительности
├── database.py       # Работа с базой данных
├── main.py           # Основной файл запуска
├── reset_db.py       # Сброс базы данных
├── utils.py          # Вспомогательные функции
└── implementations/  # Реализации парсеров
```

- Сравнение времени выполнения для разного количества URL
- Анализ эффективности каждого подхода
