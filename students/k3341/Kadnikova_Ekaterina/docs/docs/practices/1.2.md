## Описание  
Данный проект представляет собой REST API для управления данными о воинах, их профессиях и навыках. В отличие от первой практики, здесь реализовано полноценное подключение к базе данных **PostgreSQL** через **SQLModel** (обёртка над SQLAlchemy + Pydantic), с использованием **FastAPI**.

## Структура проекта  
```
.
├── main.py           # Основной файл приложения FastAPI
├── models.py         # SQLModel-модели
└── connection.py     # Подключение к БД и управление сессиями
```

db_url = 'postgresql://postgres:richard2023@localhost/warriors_db'

## Модели (`models.py`)

### Enum: `RaceType`
```python
class RaceType(Enum):
    director = "director"
    worker = "worker"
    junior = "junior"
```

### Warrior
- `WarriorBase` — базовая модель
- `Warrior` — модель таблицы с отношениями
- `WarriorResponse` — расширенный вывод с вложенными профессиями и навыками

### Profession
- `ProfessionBase` — вводимые данные
- `Profession` — модель таблицы, связанная с воинами

### Skill
- `SkillDefault` — вводимые данные
- `Skill` — модель таблицы, связанная с воинами через `SkillWarriorLink`

### SkillWarriorLink
Промежуточная таблица многие-ко-многим между навыками и воинами.

## API Эндпоинты (`main.py`)

### Инициализация
```python
@app.on_event("startup")
def on_startup():
    init_db()
```

---

### Воин

#### `POST /warrior`  
Создание нового воина.

#### `GET /warriors_list`  
Получение списка всех воинов.

#### `GET /warrior/{warrior_id}`  
Получение информации о воине по ID (с профессией и навыками).

#### `PATCH /warrior/{warrior_id}`  
Частичное обновление информации о воине.

#### `DELETE /warrior/{warrior_id}`  
Удаление воина по ID.

---

### Профессии

#### `GET /professions_list`  
Получение списка всех профессий.

#### `GET /profession/{profession_id}`  
Получение профессии по ID.

#### `POST /profession`  
Создание новой профессии.

---

### Навыки

#### `GET /skills`  
Получение списка всех навыков.

#### `POST /skills`  
Создание нового навыка.

#### `PATCH /skills/{skill_id}`  
Обновление информации о навыке.

#### `DELETE /skills/{skill_id}`  
Удаление навыка по ID.

---

### Связь Навыков и Воинов

#### `POST /skill_warriors`  
Добавление связи "воин — навык".  
Тело запроса: объект `SkillWarriorLink`:
```json
{
  "skill_id": 1,
  "warrior_id": 2
}
```
