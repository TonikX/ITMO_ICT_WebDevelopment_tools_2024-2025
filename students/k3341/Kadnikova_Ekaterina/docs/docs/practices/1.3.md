## Описание  
Данный проект представляет собой REST API для управления данными о **воинах**, их **профессиях** и **навыках**, с реализацией **моделей связей** многие-ко-многим.  
Используются библиотеки **FastAPI**, **SQLModel**, и **PostgreSQL**. В этом задании добавлены полноценные связи между таблицами (например, воины и их навыки), обновление и удаление записей, а также частичное обновление данных.

---

```
.
```

## Модели (`models.py`)

### Enum: `RaceType`
```python
class RaceType(Enum):
    director = "director"
    worker = "worker"
    junior = "junior"
```
Используется для поля `race` у воинов.

---

### Warrior

- **`WarriorBase`** — базовая модель для создания/обновления.
- **`Warrior`** — полная таблица с полем `id` и связями:
  - `profession`: один-ко-многим с моделью `Profession`.
  - `skills`: многие-ко-многим через `SkillWarriorLink`.
- **`WarriorResponse`** — модель ответа, включающая `skills` и `profession`.

---

### Profession

- **`ProfessionBase`** — входные данные (title, description).
- **`Profession`** — модель таблицы с отношением к воинам (`warriors_prof`).

---

### Skill

- **`SkillDefault`** — базовая модель для создания и обновления.
- **`Skill`** — таблица с отношением многие-ко-многим к воинам (`warriors`).

---

### SkillWarriorLink

Модель промежуточной таблицы:
```python
class SkillWarriorLink(SQLModel, table=True):
    skill_id: int
    warrior_id: int
    level: Optional[int]
```

Позволяет добавить уровень навыка конкретному воину и управлять связями.

---

### Воины

#### `POST /warrior`  
Создание нового воина.

#### `GET /warriors_list`  
Получение списка всех воинов.

#### `GET /warrior/{warrior_id}`  
Получение информации о конкретном воине (с профессией и навыками).

#### `PATCH /warrior/{warrior_id}`  
Частичное обновление информации о воине.

#### `DELETE /warrior/{warrior_id}`  
Удаление воина по ID.

---

### Профессии

#### `GET /professions_list`  
Список всех профессий.

#### `GET /profession/{profession_id}`  
Получение одной профессии по ID.

#### `POST /profession`  
Создание новой профессии.

---

### Навыки

#### `GET /skills`  
Список всех навыков.

#### `POST /skills`  
Создание нового навыка.

#### `PATCH /skills/{skill_id}`  
Обновление информации о навыке.

#### `DELETE /skills/{skill_id}`  
Удаление навыка по ID.

---

### Связи "Воин — Навык"

#### `POST /skill_warriors`  
Привязка навыка к воину.  
Ожидает объект:
```json
{
  "skill_id": 1,
  "warrior_id": 2,
  "level": 5
}
```
Проверяет существование воина и навыка, затем создаёт связь через `SkillWarriorLink`.
