# Лабораторная работа 2: Потоки, Мультипроцессинг и Асинхронность в Python

Эта лабораторная работа демонстрирует реализацию параллельной обработки с использованием трех различных подходов в Python:
1. Потоки (Threading)
2. Мультипроцессинг (Multiprocessing)
3. Асинхронность (Async/Await)

### Задача 1: Вычисление суммы
- `threading_sum.py`: Реализация с использованием потоков
- `multiprocessing_sum.py`: Реализация с использованием мультипроцессинга
- `async_sum.py`: Реализация с использованием асинхронности

### Задача 2: Парсинг веб-страниц
- `db_utils.py`: Утилиты базы данных для хранения распарсенных веб-страниц
- `threading_parser.py`: Веб-парсер с использованием потоков
- `multiprocessing_parser.py`: Веб-парсер с использованием мультипроцессинга
- `async_parser.py`: Веб-парсер с использованием асинхронности

## Запуск кода

### Задача 1: Вычисление суммы
Запустите каждую реализацию отдельно:
```bash
python threading_sum.py
python multiprocessing_sum.py
python async_sum.py
```

### Задача 2: Парсинг веб-страниц
Запустите каждую реализацию отдельно:
```bash
python threading_parser.py
python multiprocessing_parser.py
python async_parser.py
```

## Сравнение производительности

### Задача 1: Вычисление суммы
Сравнение производительности покажет:
- Потоки: Ограничены GIL, могут не показать значительного улучшения
- Мультипроцессинг: Лучшая производительность для задач, ограниченных CPU
- Асинхронность: Хорошо подходит для задач, ограниченных I/O, но может не помочь для задач, ограниченных CPU

### Задача 2: Парсинг веб-страниц
Сравнение производительности покажет:
- Потоки: Хорошо подходят для задач, ограниченных I/O, таких как веб-запросы
- Мультипроцессинг: Может иметь накладные расходы для задач, ограниченных I/O
- Асинхронность: Лучшая производительность для задач, ограниченных I/O, таких как веб-запросы
