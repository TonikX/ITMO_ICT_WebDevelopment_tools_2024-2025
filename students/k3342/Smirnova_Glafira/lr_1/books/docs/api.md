# Bookcrossing API

Приложение реализует авторизацию и регистрацию пользователей посредством JWT-токенов, CRUD API для всех 
основных моделей системы, а также функционал для поиска и управления книгами, офферами и запросами на обмен.

## Аутентификация и пользователи
* Регистрация пользователя (email, username, пароль).
* Авторизация через JWT-токен.
* Получение информации о текущем пользователе.
* Смена пароля.
* Редактирование профиля (username, возраст, информация).
* Получение списка всех пользователей (опционально с фильтром по username).
* Просмотр информации о любом пользователе по ID.

## Работа с книгами
* Создание новой книги.
* Редактирование книги (только если пользователь — первый владелец).
* Удаление книги и/или владения (если пользователь — первый владелец, удаляется книга и владение,
иначе удаляется только владение)
* Просмотр информации о конкретной книге.
* Получение списка всех книг, которыми пользователь сейчас владеет.
* Установка жанров книги списком (перезаписываются старые).
* Получение всех жанров (с ID).

## Офферы на обмен
* Создание оффера на владение (один оффер на одну активную книгу).
* Изменение комментария к офферу.
* Удаление оффера.
* Получение одного оффера по ID.
* Получение всех офферов текущего пользователя.
* Получение всех открытых офферов:
    * Поиск по названию книги или автору (без учёта регистра).
    * Фильтрация по году (от / до).
    * Фильтрация по жанру.
    * Фильтрация по пользователю.
    * Сортировка по дате создания (новые или старые сначала).

## Свопы (запросы на обмен) и сделки
* Создание свопа (запроса на обмен книгами).
* Получение всех свопов, созданных текущим пользователем (всех или только pending).
* Получение всех свопов, адресованных текущему пользователю (всех или только pending).
* Отклонение свопа.
* Принятие свопа:
    * Изменение статуса на completed.
    * Закрытие обоих офферов.
    * Завершение текущих владений.
    * Создание новых владений.
    * Создание записи о сделке.
* Получение списка всех сделок, в которых участвовал пользователь.
