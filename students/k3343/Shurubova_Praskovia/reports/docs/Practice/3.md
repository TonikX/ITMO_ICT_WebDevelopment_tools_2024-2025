# Миграции, ENV, GitIgnore и структура проекта

### Реализовать в своем проекте все улучшения, описанные в практике
Для интеграции Alembic в разрабатываемый проект установила его через пакетный менеджер.  

Сгенерировалась папка migratgions хранящая внутри себя папку с файлами миграций versions, файл окружения БД env.py и шаблон генерации миграций script.py.mako. В корне проекта добавился файл настроек alembic.ini. Создала файл миграций с помощью команды 

    alembic revision --autogenerate -m "skill added"
Применила миграции с помощью команды 
    
    alembic upgrade head
### Разобраться как передать в alembic.ini URL базы данных с помощью.env-файла и реализовать подобную передачу.
В файл .env вписала переменную, хранящую URL баззы данных. 

    DB_ADMIN = postgresql+psycopg2://postgres:89184890284p@localhost/bookcrossing_db
После установки, в файле db.py адрес БД с использованием переменных окружения указывается следующим образом:

    from dotenv import load_dotenv
    
    load_dotenv()
    db_url = os.getenv('DB_ADMIN')
